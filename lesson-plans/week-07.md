# Week 07 — Accessibility & Semantic HTML Deep Dive (Index)

This week focuses on web accessibility (a11y) principles and advanced semantic HTML for creating inclusive web experiences.

## Overview

-   Focus: Web accessibility, WCAG guidelines, ARIA, and semantic HTML best practices
-   Outcomes:
    -   Understand web accessibility importance and WCAG 2.1 guidelines
    -   Implement proper semantic HTML structure
    -   Use ARIA attributes appropriately
    -   Test and validate accessibility compliance

## Sessions

### Session 19 — Web Accessibility Fundamentals

-   Summary: Why accessibility matters, WCAG 2.1 guidelines (A, AA, AAA), types of disabilities, and assistive technologies overview.
-   Link: [week-07/session-19.md](week-07/session-19.md)

### Session 20 — Semantic HTML & ARIA

-   Summary: Semantic HTML elements in depth, landmark roles, ARIA attributes (`aria-label`, `aria-describedby`, `aria-live`), and when to use ARIA.
-   Link: [week-07/session-20.md](week-07/session-20.md)

### Session 21 — Accessible Forms & Testing

-   Summary: Accessible form design, labels, error handling, focus management, keyboard navigation, and accessibility testing tools.
-   Link: [week-07/session-21.md](week-07/session-21.md)

## Deliverables This Week

-   Complete accessibility audit of a webpage
-   Build an accessible form with proper labels and error handling
-   Test pages using screen readers and accessibility tools
-   Optional: Create an accessibility checklist for your projects
