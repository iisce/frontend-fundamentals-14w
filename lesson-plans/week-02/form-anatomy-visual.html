Understanding the Connections

The for/id Connection

The most important relationship in accessible forms:

<label for="email">Email</label>
           ‚Üì MUST MATCH ‚Üì
<input id="email" name="email" type="email" />

What this enables:

  • ‚úÖ Clicking the label focuses the input
  • ‚úÖ Screen readers announce the label when the field is focused
  • ‚úÖ Better mobile experience (larger touch target)
  • ‚úÖ Clear association for all users

The id vs name Distinction

id="email" ‚Äî For the label connection (accessibility)

name="email" ‚Äî For form submission (functionality)

üí° Tip: They often have the same value, but they serve different purposes!

Key Takeaways

  • <form> wraps the entire form
  • <fieldset> groups related fields
  • <legend> describes what the fieldset contains
  • <label for="..."> must match the input's id="..."
  • Every input needs a name attribute for form submission
  • Use appropriate type attributes (email, tel, date, etc.)
  • Always use visible labels‚Äîdon't rely on placeholders alone
