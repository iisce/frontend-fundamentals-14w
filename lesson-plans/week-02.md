# Week 02 — HTML Foundations II (Index)

This week focuses on tabular data and accessible, semantic forms.

## Overview

-   Focus: Tables and form fundamentals with accessibility built-in
-   Outcomes:
    -   Mark up tabular data with `<table>`, headers, captions
    -   Build accessible forms with `<label>`, inputs, `name`/`id`
    -   Apply HTML validation and better error UX

## Sessions

### Session 04 — Tables and Tabular Data

-   Summary: When (and when not) to use tables, anatomy (`<thead>`, `<tbody>`, `<tfoot>`), header associations with `scope` and captions.
-   Link: [week-02/session-04.md](week-02/session-04.md)

### Session 05 — Forms I — Inputs and Labels

-   Summary: Core form controls, associating labels (`for` ↔ `id`), grouping with `<fieldset>`/`<legend>`, names and values.
-   Link: [week-02/session-05.md](week-02/session-05.md)

### Session 06 — Forms II — Validation & Accessibility

-   Summary: Built-in constraints (`required`, `pattern`, `min`, `max`), help text and error messages, keyboard and screen-reader UX.
-   Link: [week-02/session-06.md](week-02/session-06.md)

## Deliverables This Week

-   Complete the table lab and form labs
-   Add validation and helpful messages to the contact/feedback form
-   Optional: Document your form’s acceptance criteria in your project README
